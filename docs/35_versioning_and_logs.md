# Versioning and Logs

Creator OS keeps version history for important creative and production artifacts.

It also records operational events for traceability and troubleshooting.

## Versioned Items

- Lyrics
- Prompt
- Gemini Judgment
- Selected Version
- Image
- Subtitle
- Render Output
- Metadata

## Logged Events

- File import
- Gemini judgment
- Winner change
- Subtitle generation
- Render started
- Render completed
- Error
- User approval
- Upload

## Rules

- Important decisions should be traceable.
- Version history should make it possible to compare previous and current states.
- Logs should include timestamps, project IDs, affected asset IDs, and concise messages.
- User approvals should be recorded as explicit events.
- Errors should include enough context for troubleshooting.
- Logs should not modify original source files.
