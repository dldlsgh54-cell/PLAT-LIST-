# Analytics Memory

Creator OS stores YouTube performance data after upload.

It also builds Creator OS Memory from user decisions, Gemini recommendations, and successful content patterns.

## Collected YouTube Data

- Views
- CTR
- Average view duration
- Likes
- Comments
- Shorts performance
- Long-to-Shorts connection performance

## Creator OS Memory

- Song styles selected by the user
- Differences between Gemini recommendations and user selections
- Common traits of successful videos
- Successful Shorts durations
- Popular BPM
- Popular moods
- Strong themes by channel

## Rules

- Analytics data should remain linked to the project, video, channel, tracks, and shorts.
- Creator OS should learn from both user choices and performance outcomes.
- Gemini recommendations and user decisions should be compared over time.
- Successful patterns should inform future recommendations.
- Memory should support channel-specific learning, not only global learning.
